# コーディング規約 (Coding Guidelines)

このプロジェクトにおけるソースコードの記述ルールです。共同開発をスムーズに進めるため、遵守をお願いします。

## 1. 基本方針
* 読みやすく、保守しやすいコードを心がける。
* 重複したコードは避け、共通化する。

## 2. 命名規則
* **変数・関数名:** キャメルケース（例: `isLoggedIn`, `getUserData`）
* **クラス名:** パスカルケース（例: `UserProfile`, `OrderManager`）
* **定数:** スネークケースの大文字（例: `MAX_RETRY_COUNT`, `API_URL`）

## 3. フォーマットとスタイル
* インデントは **スペース2つ**とします。
* 行末の不要なスペースは削除してください。
* 1行の長さは最大 **120文字** を目安とします。

## 4. コメントの書き方
* 「なぜその処理が必要なのか（Why）」を意識して書く。
* `TODO:` や `FIXME:` を活用し、今後の課題を明確にする。

## 5. レビュー前のチェックリスト
- [ ] ローカルでテストがすべて通過しているか
- [ ] 無駄な console.log やデバッグコードが残っていないか
- [ ] この規約に沿って書かれているか
